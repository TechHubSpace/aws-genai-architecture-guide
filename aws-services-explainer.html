Q: When would you use Bedrock over SageMaker?
Use Bedrock when you need a powerful LLM fast ‚Äî no training, no infra. Use SageMaker when you need a custom model trained on proprietary data or need a specialized algorithm.
Q: Why Lambda instead of ECS for this flow?
Lambda handles the inbound API call because it's event-driven, short-lived, and scales instantly with zero management. ECS hosts the persistent chat service that needs to be always available.
Q: What's the difference between ECS and EKS?
ECS is AWS's proprietary container service ‚Äî simpler, tightly integrated. EKS is managed Kubernetes ‚Äî more portable, industry-standard, better for multi-cloud or complex orchestration.
Q: How does Bedrock fit into your AI architecture?
Bedrock is the intelligence layer. It sits behind Lambda ‚Äî Lambda receives user input, sanitizes it, calls Bedrock's API (e.g. Claude), and returns the response. No model hosting needed.
Q: What are Lambda's limitations?
Max 15 min execution, no persistent state, cold start latency (~100ms‚Äì2s), 10GB memory limit. For long-running or stateful workloads ‚Üí use ECS/EKS or Step Functions.
Q: Describe the full flow for an AI chatbot.
User ‚Üí API Gateway ‚Üí Lambda (validates request) ‚Üí Bedrock (generates reply via Claude) ‚Üí optionally calls SageMaker endpoint for custom scoring ‚Üí reply served via ECS-hosted backend.
